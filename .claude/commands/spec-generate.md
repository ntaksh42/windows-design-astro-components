description: 自然言語からPC アプリ風の外部仕様書を自動生成
---

ユーザーから以下の情報を収集してください：

1. **コンポーネント名**: 生成するコンポーネントの名前（例: ScrollBar, DatePicker）
2. **説明**: コンポーネントの詳細な説明（例: 「縦横両対応のスクロールバー、マウスホイール対応、スムーズスクロール機能付き」）
3. **カテゴリー**（オプション）: ui / layout / form / feedback のいずれか（デフォルト: ui）
4. **参考コンポーネント**（オプション）: 既存コンポーネントを参考にする場合、その名前（例: Button, TextBox）

情報を収集したら、以下のコマンドを実行してください：

```bash
npm run spec:generate "<コンポーネント名>" "<説明>" [カテゴリー] [参考コンポーネント]
```

例:
```bash
npm run spec:generate "ScrollBar" "縦横両対応のスクロールバー、マウスホイール対応"
npm run spec:generate "DatePicker" "日付選択コンポーネント" "form" "ComboBox"
```

## 実行後の手順

1. **生成された仕様書を確認**
   - ファイル: `src/pages/<コンポーネント名>-demo.mdx`
   - 内容をレビューし、必要に応じて修正

2. **生成内容をユーザーに報告**
   - 生成された仕様書のパスを伝える
   - 主要なセクション（概要、プロパティ、バリエーション等）の内容を簡単に要約
   - 次のステップ（コンポーネント実装）を提案

3. **オプション: 生成された仕様書の一部を表示**
   - 概要セクションを読んで、ユーザーに見せる
   - プロパティ一覧を確認

## 重要な注意事項

- コマンド実行前に、必要な情報がすべて揃っているか確認
- 説明は具体的であるほど、高品質な仕様書が生成される
- 生成後は必ず結果をユーザーに報告
- 必要に応じて、生成された仕様書の修正を支援

## 仕様書のフォーマット

生成される仕様書には以下のセクションが含まれます：

1. 概要（目的、対象ユーザー、主要機能）
2. 外観仕様（基本デザイン、デザイントークン）
3. プロパティ仕様（一覧表、詳細説明）
4. バリエーション仕様
5. 状態仕様（状態遷移、各状態の外観）
6. 動作仕様（ユーザー操作、キーボード操作、イベント）
7. 実装例（基本、実用例、よくあるパターン）
8. アクセシビリティ（ARIA属性、キーボードナビゲーション）
9. 制約事項・注意事項
10. 参考情報
11. 付録: 実装メモ（CSS、TypeScript）
