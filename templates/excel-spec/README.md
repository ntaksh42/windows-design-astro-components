# Excel仕様書テンプレート

このディレクトリには、Excelメモから仕様書を生成するためのテンプレートが含まれています。

## 📋 テンプレートファイル

- `DatePicker-sample.csv` - CSVフォーマットのサンプル（Excelで開けます）

## 🎯 使い方

### 1. テンプレートをコピー

```bash
# CSVテンプレートをコピー
cp templates/excel-spec/DatePicker-sample.csv ./my-component.csv
```

または、Excelで直接 `DatePicker-sample.csv` を開いて編集してください。

### 2. 内容を編集

以下の項目を自分のコンポーネントに合わせて編集：

- **コンポーネント名**: 作成するコンポーネントの名前
- **説明**: コンポーネントの詳細な説明
- **カテゴリー**: ui / layout / form / feedback
- **プロパティ**: 各プロパティの情報
- **機能**: 主要機能のリスト
- **使用例**: コード例
- **備考**: 注意事項など

### 3. 仕様書を生成

```bash
npm run spec:from-excel ./my-component.csv
```

または Claude Code のスラッシュコマンドで：

```
/spec-from-excel
```

## 📝 テンプレートフォーマット

### 縦型フォーマット（推奨）

```csv
項目,値,値2,値3
コンポーネント名,DatePicker,,
説明,日付選択コンポーネント,,
カテゴリー,form,,
,,,,
## プロパティ,,,
minDate,Date,2000-01-01,選択可能な最小日付
maxDate,Date,2100-12-31,選択可能な最大日付
```

**特徴:**
- 1行目: 項目名
- 2行目: 値
- セクションは `##` で区切る
- プロパティは複数列で詳細を記述

### 横型フォーマット

Excelで表として管理したい場合：

| コンポーネント名 | 説明 | カテゴリー |
|----------------|------|-----------|
| DatePicker | 日付選択コンポーネント | form |

**特徴:**
- 1行目: ヘッダー
- 2行目以降: データ
- シンプルだが、詳細情報は少なめ

## 🎨 セクション説明

### 必須セクション

#### コンポーネント名
```
コンポーネント名,DatePicker
```

コンポーネントのPascalCase名を記載。

#### 説明
```
説明,日付選択コンポーネント、カレンダーUI付き
```

コンポーネントの機能を簡潔に説明。

### オプションセクション

#### カテゴリー
```
カテゴリー,form
```

- `ui`: ボタン、ラベル等
- `layout`: コンテナ、グリッド等
- `form`: 入力要素
- `feedback`: アラート、トースト等

#### ## プロパティ
```
## プロパティ,,,
プロパティ名,型,デフォルト値,説明
minDate,Date,2000-01-01,選択可能な最小日付
maxDate,Date,2100-12-31,選択可能な最大日付
```

各プロパティを4列で記述：
1. プロパティ名
2. 型（string, number, boolean, Date等）
3. デフォルト値
4. 説明

#### ## 機能
```
## 機能,,,
機能1,カレンダーUIでの日付選択
機能2,キーボード入力にも対応
```

主要機能をリストアップ。

#### ## 使用例
```
## 使用例,,,
例1,<DatePicker />
例2,<DatePicker minDate="2024-01-01" />
```

実際のコード例を記載。

#### ## 備考
```
## 備考,,,
備考,IEは非対応です
```

注意事項や制約を記載。

## 💡 Tips

### Excelで効率的に編集する方法

1. **セルの結合を避ける**: セルを結合すると読み込みが複雑になります
2. **空行でセクションを区切る**: 見やすくなります
3. **列幅を調整**: 長い説明文も読みやすく
4. **フィルター機能を活用**: プロパティが多い場合に便利

### よくある失敗と対処法

#### ❌ コンポーネント名が抽出されない
→ 1列目に「コンポーネント名」「Name」「Component」のいずれかを記載

#### ❌ プロパティが抽出されない
→ プロパティセクションの前に `## プロパティ` を記載

#### ❌ 日本語が文字化けする
→ CSVをUTF-8で保存（Excelの「名前を付けて保存」→「CSV UTF-8」を選択）

## 🚀 実践例

### シンプルな例（最小限）

```csv
コンポーネント名,ToolTip
説明,マウスオーバーで表示されるツールチップ
```

これだけでも仕様書が生成されます！

### 詳細な例（フル活用）

`DatePicker-sample.csv` を参照してください。
プロパティ、機能、使用例、備考をすべて記載した完全な例です。

## 📦 複数コンポーネントの管理

1つのExcelファイルに複数のシートを作成して、コンポーネントごとに管理できます：

```bash
# シートを指定して生成
npm run spec:from-excel ./components.xlsx "DatePicker"
npm run spec:from-excel ./components.xlsx "TimePicker"
npm run spec:from-excel ./components.xlsx "DateTimePicker"
```
